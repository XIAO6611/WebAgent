## 基于多模态大模型的网页自动操作 Agent：

## 面向旅游场景的智能决策助手系统设计与实现

------

### 一、研究背景与研究意义

随着人工智能技术的快速发展，大语言模型（Large Language Model, LLM）和视觉语言模型（Vision-Language Model, VLM）在自然语言理解、图像识别以及复杂任务推理等方面取得了显著进展。近年来，研究者逐渐将大模型从传统的文本生成任务扩展到更复杂的交互式应用场景，例如自动化办公、智能客服以及智能搜索系统等。其中，一个重要的发展方向是构建能够理解网页界面并自动执行浏览器操作的智能系统，即**网页智能体（Web Agent）**。

传统互联网应用中，用户在完成一个复杂任务时通常需要手动浏览多个网站。例如，在规划一次旅行时，用户需要访问多个平台，如酒店预订网站和社交媒体平台，对不同酒店进行价格比较、评分筛选以及用户评价阅读。这一过程不仅耗时较长，而且需要用户具备一定的信息检索能力。因此，开发一个能够自动执行这些操作并给出合理建议的智能系统具有重要的实际应用价值。

Web Agent 是近年来人工智能领域的研究热点之一。与传统的爬虫系统不同，Web Agent 不仅能够抓取网页数据，还能够理解网页界面结构，并根据用户目标动态生成操作序列，例如点击按钮、输入文本以及滚动页面等。这种系统更加接近人类用户的浏览行为，具有更高的智能性和灵活性。随着多模态大模型的发展，利用网页截图作为输入，通过视觉语言模型识别页面元素并生成操作指令，已经成为当前 Web Agent 研究的重要方向。

在旅游领域，用户往往需要综合多个信息来源来做出决策。例如，酒店预订网站可以提供价格、评分以及位置等结构化信息，而社交媒体平台（如小红书）则可以提供用户真实评价和旅行体验。将这两类信息结合，可以显著提升推荐结果的质量。因此，本项目以旅游场景为应用背景，设计并实现一个基于多模态大模型的网页自动操作 Agent，实现跨网站信息获取与智能决策支持。

本研究不仅具有一定的工程实践意义，同时也能够帮助学生深入理解多模态模型、自动化浏览器技术以及智能体系统设计等关键技术，对提升人工智能系统开发能力具有重要价值。

------

### 二、研究目标与任务定义

本项目的主要目标是设计并实现一个基于多模态大模型的网页自动操作 Agent 系统，使其能够理解用户输入的自然语言任务，并在真实网页环境中自动执行浏览器操作，从而完成复杂的旅游信息检索与酒店推荐任务。

具体而言，本系统需要实现以下核心功能：

1）理解用户输入的自然语言任务，例如：

帮我找成都春熙路附近价格 500 元以内、评分较高的酒店，并参考小红书用户评价给出推荐。

2）自动访问酒店预订网站（携程），执行搜索和筛选操作，包括：

输入城市名称
 设置价格范围
 按照评分排序
 打开酒店详情页面

3）自动访问社交媒体网站（小红书），执行信息检索操作，包括：

搜索酒店名称
 打开用户帖子
 读取用户评价

4）整合来自不同网站的信息，生成推荐结果并返回给用户。

为了满足课程对多模态系统的要求，本项目将使用网页截图作为视觉输入，并利用视觉语言模型识别页面内容，从而实现对网页界面的理解和操作决策。

系统的整体工作流程如下：

用户输入任务
 → Agent 分析任务目标
 → 打开浏览器访问目标网站
 → 获取网页截图
 → 视觉模型识别页面信息
 → 语言模型生成操作指令
 → 浏览器执行操作
 → 输出最终结果

### 三、相关工作调研

近年来，随着大模型技术的发展，Web Agent 系统已经成为人工智能研究的重要方向。国内外研究机构和企业相继提出了多种基于大模型的网页自动操作系统。以下介绍几个具有代表性的研究工作。

------

#### 1. WebGPT

WebGPT 是早期基于大语言模型的网页浏览系统之一。该系统通过将网页信息转换为文本形式，使语言模型能够在浏览器环境中执行搜索、点击和阅读等操作，从而完成复杂的信息检索任务。WebGPT 的研究表明，大语言模型可以通过逐步决策的方式完成多步骤任务，为后续 Web Agent 研究奠定了基础。

------

#### 2. WebVoyager

WebVoyager 是近年来提出的一种基于视觉信息的网页智能体系统。与传统方法不同，该系统直接使用网页截图作为输入，通过视觉语言模型识别页面内容，并生成下一步操作指令。例如，在购物网站中，系统可以识别“加入购物车”按钮的位置，并自动点击该按钮。WebVoyager 的研究证明，视觉信息可以有效提升网页操作的准确性。

------

#### 3. BrowserAgent

BrowserAgent 是一种结合大语言模型和浏览器自动化技术的智能体系统。该系统通过语言模型生成浏览器操作指令，并利用自动化工具执行这些操作，例如点击按钮和输入文本。BrowserAgent 采用模块化设计，将任务规划和操作执行分离，从而提高了系统的稳定性和可扩展性。

------

#### 4. OpenClaw

OpenClaw 是一个开源 Web Agent 框架，支持在真实浏览器环境中运行智能体。该系统提供了一套完整的工具链，用于网页截图获取、操作执行以及任务记录。OpenClaw 的设计理念强调系统的可复现性和可扩展性，是当前 Web Agent 研究中具有代表性的工程实现之一。

------

#### 5. Tabbit（美团）

Tabbit 是由美团开发的一种面向本地生活服务的智能浏览器助手。该系统能够自动执行订餐、订酒店以及查询信息等任务，并根据用户需求提供个性化推荐。Tabbit 的成功应用表明，Web Agent 技术在实际商业场景中具有广阔的发展前景。

------

### 四、系统总体方案设计

本系统采用模块化架构设计，将整个智能体系统划分为多个功能模块，以提高系统的可维护性和可扩展性。系统总体结构如图所示（在正式报告中可绘制系统架构图）。

系统主要由以下四个核心模块组成：

1）感知模块（Perception Module）
 2）任务规划模块（Planning Module）
 3）操作执行模块（Execution Module）
 4）状态管理模块（Memory Module）

------

#### 1. 感知模块（多模态处理模块）

感知模块负责获取网页界面信息，并将其转换为可供模型处理的数据。该模块的主要输入为网页截图，输出为页面元素信息，例如按钮位置、文本内容以及价格信息。

具体实现步骤如下：

首先，系统使用浏览器自动化工具对当前网页进行截图。
 然后，将截图输入视觉语言模型进行分析。
 最后，将识别结果传递给任务规划模块。

该模块是实现多模态能力的关键部分，也是本系统区别于传统爬虫系统的重要特征。

------

#### 2. 任务规划模块

任务规划模块负责根据用户输入生成操作步骤。该模块的核心是大语言模型，其主要功能包括：

理解用户需求
 分析当前页面状态
 生成下一步操作指令

例如，当用户输入：

帮我找成都 500 元以内的酒店

系统可能生成以下操作步骤：

点击搜索框
 输入成都
 点击搜索按钮
 选择价格筛选条件
 按评分排序

------

#### 3. 操作执行模块

操作执行模块负责将模型生成的操作指令转换为实际的浏览器行为。该模块通过浏览器自动化工具实现，例如：

点击按钮
 输入文本
 滚动页面
 打开链接

在本项目中，计划使用 Playwright 作为浏览器自动化工具，因为其稳定性较高，并且支持多种浏览器环境。

------

#### 4. 状态管理模块

状态管理模块用于记录系统当前状态以及历史操作信息，包括：

当前网页地址
 已执行操作列表
 任务完成情况

该模块可以帮助系统避免重复操作，并在发生错误时进行恢复。

------

### 六、实验设计与评测方案

为了验证系统的有效性，本项目将设计一系列实验任务，并使用定量指标对系统性能进行评估。

------

#### 1. 实验网站

本项目选择两个真实网站作为实验环境：

携程（酒店搜索）
 小红书（用户评价）

------

#### 2. 任务设计

每个网站设计三个任务。

------

携程任务：

任务 1：查找价格 500 元以内的酒店
 任务 2：按评分排序酒店
 任务 3：打开酒店详情页面

------

小红书任务：

任务 1：搜索成都酒店推荐
 任务 2：打开一条帖子
 任务 3：提取关键词

------

#### 3. 实验规模

按照课程要求，每个任务进行 10 次独立测试。

实验总规模为：

2 个网站
 × 3 个任务
 × 10 次测试

共计：

60 次实验

------

#### 4. 评测指标

本项目将使用以下指标评估系统性能。

------

成功率（Success Rate）

成功完成任务的次数
 除以
 总任务次数

------

平均操作步数（Average Steps）

系统完成任务所需的平均操作数量

------

执行时间（Execution Time）

系统完成任务所需的平均时间

------

错误类型分析（Error Analysis）

识别错误
 规划错误
 执行错误

### 七、可行性分析

------

#### 1. 技术可行性

本项目所需技术均为成熟技术，包括：

浏览器自动化技术

 内存：16GB
 硬盘：10GB

第 1 周：完成开题报告

 第 4 周：实现截图功能
 第 5 周：实现多模态识别
 第 6 周：系统集成
