### 一、 Chrome Manifest V3 架构与底层限制问题

| 痛点现象 | 根本原因 | 解决方案 |
| :--- | :--- | :--- |
| **`window is not defined` 报错导致崩溃** | 在 MV3 架构下，`background.js` 变成了 Service Worker，它运行在后台独立线程，没有 DOM 树和 `window`/`document` 对象。 | 严格分离职责。将所有需要获取视口尺寸、DOM 元素的逻辑下放到 `content.js` 中，通过 `chrome.tabs.sendMessage` 进行异步通信获取数据。 |
| **控制台显示 Service Worker (无效)** | 这是 Chrome 优化内存的底层机制。当插件闲置时，Service Worker 会自动进入“休眠”状态。 | 确认这是正常现象。只要通过 popup 点击或消息通信唤醒，它会瞬间复活接管任务，无需做额外的代码修复。 |
| **无法提取 DOM，或报错“在系统页面执行”** | 出于极高的安全限制，Chrome 严禁插件在 `chrome://`、`edge://` 及新标签空白页注入脚本或截图。 | 增加前置拦截器（Security Guard）。在 `background.js` 启动前检查 `activeTab.url`，若为受限页面则抛出异常并终止；规范测试流程，必须在真实加载完毕的网页（如 Baidu）刷新后测试。 |

### 二、 大模型 API 调用与工程化处理

| 痛点现象 | 根本原因 | 解决方案 |
| :--- | :--- | :--- |
| **返回 `1210 参数有误` 且调用失败** | 将截图（`image_url`）传给了一个纯文本大模型（如 `glm-4.5-air`），导致接口校验失败。 | **模型能力解耦（双模型架构）**：阶段一使用多模态视觉模型（`glm-4.6v`）负责看图和坐标决策；阶段二使用低成本、强逻辑的纯文本模型（`glm-4.5-air`）负责非结构化文本的总结和 Markdown 生成。 |
| **后台静默报错，前台面板一直卡在启动中** | 弹窗（Popup）失去焦点后会被销毁；且后台若未向 Popup 发送完成状态，UI 会陷入死等。 | 1. 引入独立的全屏 `options.html` 安全存储 API Key。2. 在后台使用 `try...catch` 包裹主循环，封装统一的 `sendLog` 函数，将实时状态或报错反向推送至前端。 |
| **模型返回的动作指令包含 Markdown 标签解析失败** | 大模型倾向于输出被 ` ```json ` 包裹的代码块，导致原生的 `JSON.parse` 直接抛出异常中断运行。 | 引入 **JSON 正则清洗机制**（`content.match(/\{[\s\S]*\}/)`），强制从杂乱的回复中提取出合法的 JSON 字符串再进行解析。 |

### 三、 DOM 物理交互与感知

| 痛点现象 | 根本原因 | 解决方案 |
| :--- | :--- | :--- |
| **Agent 可以点击网页，但无法在搜索框中打字** | 大模型的动作空间未定义输入行为；且直接给 Input 赋 `value` 无法触发现代前端框架（React/Vue）的数据双向绑定。 | 1. 扩展大模型动作空间，新增 `type` 和 `press_enter`。2. 在 `content.js` 中模拟物理级输入：赋值后必须强制派发 `Event('input')` 和 `Event('change')`，并通过模拟 `KeyboardEvent` 触发回车动作。 |
| **纯视觉大模型给出的坐标点击经常出现偏移** | 大模型直接根据截图估算绝对像素坐标容易产生“视觉幻觉（Hallucination）”。 | 采用 **“视觉为主，轻量 DOM 为辅”** 策略。由 `content.js` 提前扫描页面所有 `<button>`/`<input>`，计算精准中心坐标，连同截图一并喂给大脑作为参考。 |

### 四、 AI Agent 认知与控制系统 (核心亮点)

| 痛点现象 | 根本原因 | 解决方案 |
| :--- | :--- | :--- |
| **指令中的个人信息无法准确映射到本地知识库** | 大模型语义检索能力在面对模糊指令时存在不确定性，把变量当成了字符串实体。 | 引入 **占位符注入（Template Grounding）**。在下发任务前，在后台正则遍历 `{{key}}` 符号，直接将知识库数据替换进系统指令中。 |
| **大模型完成任务后不停止，陷入无限瞎点的死循环** | 模型缺乏“全局任务进度感”，不知道何时该终止，直到触发了最大步数安全锁。 | 引入 **ReAct（推理与行动）状态机架构**。强制模型在输出动作前，必须先输出 `thought`（当前思考）并维护一个 `remaining_plan`（剩余计划表）。通过划掉已完成步骤，明确其判定 `done` 的退出条件。 |
